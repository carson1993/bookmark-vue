# 📦 浏览器扩展安装指南

## 前提条件

- Chrome 浏览器 (推荐最新版本)
- 或 Edge 浏览器
- 或其他基于 Chromium 的浏览器

## 🛠️ 安装步骤

### 方法一：从 bookmark-app/dist 目录安装（推荐）

1. **构建项目**
   ```bash
   cd bookmark-app
   npm run build
   ```

2. **打开浏览器扩展管理页面**
   - Chrome: `chrome://extensions/`
   - Edge: `edge://extensions/`

3. **启用开发者模式**
   - 在右上角找到"开发者模式"开关
   - 将其切换为"开启"状态

4. **加载扩展**
   - 点击"加载已解压的扩展程序"按钮
   - 选择 `bookmark-app/dist` 目录
   - 点击"选择"按钮

5. **验证安装**
   - 扩展应该出现在扩展列表中
   - 查看是否有错误提示
   - 点击新标签页测试功能

### 方法二：从根目录安装

如果直接使用根目录的文件：

1. 打开浏览器扩展管理页面
2. 启用开发者模式
3. 加载已解压的扩展程序
4. 选择 `bookmark` 根目录
5. 点击"选择"按钮

## 🔧 常见问题

### Q1: 扩展无法加载？

**A:** 检查以下几点：
- `manifest.json` 文件是否存在
- 所有引用的文件路径是否正确
- 控制台是否有错误信息

### Q2: 书签没有显示？

**A:** 
- 检查浏览器是否有书签
- 确认扩展有"书签"权限
- 刷新页面重试

### Q3: 图标显示异常？

**A:**
- 检查 `png` 和 `gif` 文件夹是否存在
- 确认文件路径正确
- 清除浏览器缓存

### Q4: 如何更新扩展？

**A:**
1. 修改代码后重新构建：`cd bookmark-app && npm run build`
2. 在扩展管理页面点击"刷新"图标
3. 或者移除扩展后重新加载

### Q5: 如何调试？

**A:**
1. 在扩展管理页面点击"检查视图" → "background page"
2. 打开开发者工具查看日志
3. 检查控制台是否有错误

## 📝 开发者模式

### 实时开发

如果需要实时开发：

```bash
# 在 bookmark-app 目录
npm run dev

# 然后在浏览器中加载源代码目录
# 注意：需要配置 Vite 为开发模式
```

### 代码热更新

当前配置需要手动构建：
```bash
cd bookmark-app
npm run build
```

如需热更新，可以配置 Vite 的开发服务器。

## 🎨 自定义配置

### 修改扩展名称

编辑 `manifest.json`：
```json
{
  "name": "你的扩展名称",
  "description": "你的扩展描述"
}
```

### 修改版本号

```json
{
  "version": "1.0.0"
}
```

### 修改图标

替换 `png/pkq1.png` 文件即可。
